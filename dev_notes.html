My Development Notes

Foundations and Design Strategy (AC 1.1 - 2.3)

I built this site using HTML5 for the structure and CSS3 for the minimalist design (AC 1.1). I used semantic tags like nav, main, and section to organize the content. For simple formatting (AC 1.2), I used bold tags to highlight key terms and kept the layout clean with plenty of white space. I used unordered lists for the navigation menu and ordered lists for scientific facts (AC 1.3). By using a single style.css file (AC 1.4), I ensured that every page has the same professional, simplistic look.

Design Purpose and Structure

The main goal of the site is to provide clear information about the climate crisis to students and the general public (AC 2.1). I chose a Flat Structure for the navigation so that all 10 pages are immediately accessible from the top bar (AC 2.3). This is much better than a deep sub-menu structure because it allows users to find information quickly without getting lost. I also set the width to 80% to make the text easier to read on wide screens.

Execution and Standards (AC 3.1 - 4.2)

To make the site interactive (AC 3.1), I implemented a Climate Action Survey using FormSubmit. I simplified the form by removing all borders and legends to match the site's minimalist theme. I thoroughly tested the form (AC 3.2) to ensure that the "required" attributes prevent empty submissions. Based on my testing (AC 3.3), I removed the top header and the grey background boxes to make the site load faster and look more modern on mobile devices.

Professional Workflow and Ethics

I wrote all the code manually using the Vim editor and used Git to track every change I made (AC 3.4). This professional workflow allowed me to experiment with different layouts and revert back whenever I encountered a bug. For technical standards (AC 4.1), I ensured the site is lightweight and fast by optimizing the image sizes. Legally (AC 4.2), I have only used public domain or Creative Commons images and provided a full bibliography to respect copyright laws.
