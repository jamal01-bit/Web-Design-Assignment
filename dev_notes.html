HTML & CSS Foundations

I built this site using HTML5 for the structure and CSS3 for the design. HTML is the skeleton of a website, and I used different tags to define the text and images. For simple formatting, I used bold tags to make text stand out. The navigation uses list items and links to connect all 10 pages together.

By using a single style.css file, I could control the design of every page at once. This makes the site look professional because every page has the same colors and fonts. I also used margin: auto to center the site.

Design Strategy & Purpose

The main goal of the site is to teach the general public about climate change and why it is urgent. When I was planning, I chose a Flat Structure where every page is just one click away. I chose this because it is much easier for users to find what they need quickly. I also set the content width to 80% so the text lines are not too long on big screens.

Professional Workflow

I wrote all the code by hand to learn how it actually works instead of using drag-and-drop tools. I used Git to save my progress and track every change. This helped me fix errors in my links by checking what I had changed previously. Saving my work meant I could always go back to an older version if I broke something.

Interactivity & Testing

To make the site interactive, I added a Climate Survey form. I checked that the required boxes worked so people couldn't send empty forms. I also looked at the site on different screen sizes to make sure it looked good on both mobile and desktop. Based on my testing, I simplified the navigation menu to ensure it remains easy to read.

Legal & Technical Standards

I wanted the site to be fast and safe. I kept it fast by not using too many heavy images that slow down the loading time. For legal reasons, I only used images that are free to use, and I made sure to list my sources like NASA in the bibliography. This means I am not breaking any copyright laws.
