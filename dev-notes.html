HTML & CSS Foundations (AC 1.1 - 1.4)

I built this site using HTML5 for structure and CSS3 for design. Key tags like h1, p, and ul organize content, while a href links all 11 pages and external resources like NASA (AC 1.3). I used br for spacing and b for emphasis (AC 1.2). By using a central style.css file, I maintained a consistent professional look across the entire site, using classes for reusable elements like the "highlight" boxes and "page-img" layouts (AC 1.4).

Design Strategy & Purpose (AC 2.1 - 2.3)

The site aims to educate the general public about the climate crisis. I initially considered a hierarchical "Deep Structure," but chose a Flat Structure (AC 2.3). This ensures users can access any topic in one click, which is vital for an awareness campaign. I also avoided outdated framesets in favor of a modern block layout with 80% widths for better readability (AC 4.1).

Professional Workflow (AC 3.4)

I used a professional developer's workflow, writing all code manually in the Vim editor via the Linux terminal. This deepened my understanding of semantic HTML. For version control, I used Git to track changes and handle troubleshooting. For example, when resolving a 404 error on the survey redirect, I used Git to compare versions and updated the URL to the canonical lowercase format required by GitHub Pages.

Interactivity & Testing (AC 3.1 - 3.3)

The "Take Action" page features an interactive survey. I integrated FormSubmit to handle data without needing a back-end server. Testing revealed that GitHub Pages redirects are case-sensitive, so I adjusted the _next URL and disabled the CAPTCHA for a smoother user experience (AC 3.3). Automated HTML5 validation ensures all required fields are filled before submission (AC 3.2).

Legal & Technical Standards (AC 4.1, 4.2)

To ensure performance, I used lightweight CSS animations, such as the blinking alert (AC 1.2) to highlight urgency. Legally, I used public domain or Creative Commons images and cited experts like David Attenborough and NASA in my references (AC 4.2). The site is designed for fast loading and follows GDPR principles by only collecting necessary survey names.
