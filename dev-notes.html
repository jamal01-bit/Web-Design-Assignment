HTML & CSS Foundations (AC 1.1 - 1.4)

I built this site using HTML5 for structure and CSS3 for design. Semantic tags like header, nav, main, and footer provide a clear hierarchy for accessibility. For text formatting (AC 1.2), I used b for emphasis and br for intentional line breaks. The navigation (AC 1.3) uses a href tags to link all 11 pages into a cohesive network, including an external link to NASA's climate data for scientific credibility. By using a central style.css file (AC 1.4), I separated the "look and feel" from the content, allowing me to update colors, fonts, and borders across the entire site instantly. This "Cascading" nature of CSS ensures that every page feels like part of the same professional project.

Design Strategy & Purpose (AC 2.1 - 2.3)

The site's primary goal is to educate the general public about the urgency of the climate crisis (AC 2.1). During the planning phase, I evaluated two layouts: a "Deep Structure" with many sub-menus and the chosen Flat Structure (AC 2.3). I decided on the flat structure because it offers "one-click" access to all topics, which is crucial for keeping users engaged. I specifically designed the main content area with an 80% width (AC 4.1) to prevent long text lines, which can be tiring to read on large monitors. This user-centric approach ensures that students and citizens can find and digest information quickly and efficiently.

Professional Workflow (AC 3.4)

I adopted a senior developer's workflow by writing all code manually in the Vim editor within the Linux terminal. This helped me master HTML syntax without relying on "drag-and-drop" tools. For version control, I used Git to track every change. This was invaluable when troubleshooting a 404 error on the survey redirect. By using git diff, I identified that the URL case was the issue and corrected it to the canonical lowercase format required by GitHub Pages. Regularly committing my work allowed me to experiment safely, knowing I could always revert to a stable version of the site if something broke.

Interactivity & Testing (AC 3.1 - 3.3)

To make the site interactive (AC 3.1), I built a "Climate Survey" using a standard HTML form. I integrated FormSubmit to process these results, which allowed me to have a working form without setting up a private database or server. Testing was a major part of this project (AC 3.2). I verified that HTML5 "required" attributes correctly blocked empty submissions. Based on testing results (AC 3.3), I disabled the "I am not a robot" CAPTCHA and updated the _next redirect URL to ensure a seamless transition to the "Thank You" page after submission. This iterative testing process directly improved the site's overall reliability and user experience.

Legal & Technical Standards (AC 4.1, 4.2)

Performance and security were top priorities (AC 4.1). I used a lightweight blinking animation (AC 1.2) to draw attention to the most urgent climate alerts without slowing down page load times. From a legal standpoint (AC 4.2), I was careful to only use public domain or Creative Commons images, citing sources like Sir David Attenborough and NASA in my bibliography. This ensures the project is copyright-compliant. I also respected privacy standards by ensuring the survey only collects the minimum data needed, following the core principles of GDPR.
