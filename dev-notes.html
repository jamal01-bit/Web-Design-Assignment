HTML Basics (AC 1.1, 1.2, 1.3)

To create this website, I used HTML (HyperText Markup Language) which is the standard language used to create webpages. HTML uses "tags" to tell the browser how to display different parts of the content. For example, the h1 tag is used for the main title, and the p tag is used for paragraphs.
I also used the a href tag for hyperlinks (AC 1.3) which allows users to navigate between my 11 pages and also visit external links like NASA.
To format the text properly (AC 1.2), I used br for line breaks where I needed extra distance between words. I also used b to make important words stand out. These basic tags are the foundation of any website.

Purpose and My Choice of Topic (AC 2.1)

I decided to build my website on climate change because it is a topic I feel strongly about and it is always in the news lately. My main goal was to create an educational resource that explains the problem simply but also gives people a way to see how they can help. I wanted the site to be easy for anyone to use, whether they are a student like me or just someone worried about the planet. My target audience is the general public, but I kept the language simple so it is accessible for younger students too. By adding the 'Climate Quiz' page, I hoped to move the site beyond just providing information and actually encourage people to test what they have learned from the other pages.

Website Design and Alternative Evaluation (AC 2.2, 2.3)

While planning the site, I had to think about how users would find their way around. I looked at two different ideas. The first design used a 'Deep Hierarchy' structure where you had to click through several layers to find information. I thought this might be too confusing, so I went with a 'Flat' site structure instead. This means every page is just one click away from the main menu at the top. I think this is a better choice because it makes the site feel much more open and easier to use. Users can find facts quickly without getting lost in too many sub-menus. This layout also made it easier for me to manage all 11 pages and keep the design consistent.

Professional Workflow: Vim and Git (AC 3.4)

To build this project, I used a developer's workflow. Instead of a standard visual editor, I wrote all my HTML and CSS code using the Vim text editor directly in the Linux terminal. This helped me understand the code better because I had to type every tag manually. To manage my progress, I used Git for version control. I initialized a local repository and regularly committed my changes to keep track of my work. Once the site was ready, I pushed the entire project to GitHub. This allowed me to store my code securely and provides a professional way to show my technical skills to my teacher.

The Role of CSS (AC 1.4)

I used Cascading Style Sheets (CSS) to separate the design of the website from the content. This is a very powerful technique because it allowed me to change the look of all 11 pages just by editing one single file (style.css).
CSS improved the look of my website by allowing me to add background colors, change fonts, and add borders to my headings. It also allowed me to create a "sticky" navigation bar that stays at the top of the screen as you scroll. Without CSS, the website would just be a list of plain text and links, which would not be very engaging for the user. I also added a blinking effect (AC 1.2) using CSS animations to highlight the urgency of the climate crisis on certain pages.

Performance, Security, and Legal Issues (AC 4.1, 4.2)

I built the site using HTML5 and CSS3. I used a standard block layout and percentage-based widths (80%) to make sure the text is readable on different screens. I avoided heavy animations so the pages load fast, which is important for performance. For the quiz form, I used pure HTML5 validation with the 'required' attribute to keep the data clean. Legally, I was careful with images and facts. I used public domain images and credited sources like NASA and National Geographic to avoid copyright problems. I also ensured the site follows basic GDPR ideas by only asking for the name and email needed for the quiz.
