Purpose and My Choice of Topic (AC 2.1)

I decided to build my website on climate change because it is a topic I feel quite strongly about and it's always in the news lately. My main goal for the site was to create an educational hub that explains the problem simply but also gives people a way to get involved. I wanted the site to be easy for anyone to use, whether they are a student like me or just someone worried about the planet. My target audience is basically the general public, but I tried to keep the language simple so it's accessible for younger students too. By adding the 'Take Action' page, I hoped to move the site beyond just providing information and actually encouraging people to join a movement or sign a petition.

Website Performance and Navigation (AC 4.1)

When I was designing the layout, I spent quite a bit of time researching how to make a menu that stays at the top of the screen when you scroll down. I found out about 'sticky' positioning in CSS, which I think makes the site much easier to navigate because you don't have to scroll all the way back up to change pages. I chose a 'flat' site structure, which means all the pages are just one click away from each other. I think this is better for performance because it reduces the number of clicks a user has to make to find what they want. I also tried to keep my code clean and didn't use any big external libraries, which helps the pages load much faster, especially if someone is looking at the site on their phone or a slow internet connection.

Security, Legal Issues, and Challenges (AC 4.2)

One of the biggest challenges I had was the 'Take Action' form. Since we haven't covered much Javascript in class yet, I decided to stick to pure HTML5 for the form. I used the 'required' attribute on the input tags to make sure people can't submit an empty form, which helps with data integrity. For security, I researched how a real site would protect user data, and I've mentioned in my notes that a live site would need an SSL certificate (HTTPS) to keep emails safe. Legally, I had to be careful with the images and facts I used. I made sure to use public domain images from places like Wikimedia and I've tried to credit my sources, like NASA, to avoid any copyright problems. I also made sure the site follows basic GDPR ideas by only asking for the info we actually need for the petition.

Tools and Techniques (AC 3.4)

I built the whole site using HTML5 and CSS3. I used a separate CSS file because it's much easier to manage the design of 11 pages from one place. I used Flexbox for the navigation bar because it's really good for making sure the menu looks right on different screen sizes. Overall, I think I've met all the project requirements by building a functional, 11-page interactive site that is both educational and easy to use. The process of testing the links on different browsers helped me find a few small bugs in my CSS which I fixed before the final submission.
